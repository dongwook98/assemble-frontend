# 그룹 목록 및 검색 (Group List & Search) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 다양한 필터(카테고리, 레벨, 지역 등)와 검색어를 사용하여 모임 목록을 탐색하고 조회하는 기능.
*   **비즈니스 목적**: 사용자가 본인의 관심사와 조건에 맞는 모임을 쉽고 빠르게 발견하여 커뮤니티 참여를 유도함.
*   **성공 지표 (KPI)**: 검색 및 필터 이용률, 모임 상세 페이지 진입률.

## 2. 사용자 스토리 (User Stories)
*   "사용자는 특정 카테고리(스터디, 운동 등)의 모임만 모아보고 싶어 한다."
*   "사용자는 본인의 수준(입문, 중급 등)에 맞는 모임을 필터링하여 찾고 싶어 한다."
*   "사용자는 온/오프라인 여부나 특정 지역의 모임을 선택하여 보고 싶어 한다."
*   "사용자는 최신순이나 인기순 등으로 모임을 정렬하여 보고 싶어 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 카테고리 탭을 통한 대분류 필터링 (전체, 스터디, 운동 등)
*   [x] 다중 선택 가능한 세부 필터링 (참여 방식, 레벨)
*   [x] 필터 초기화 기능
*   [x] 정렬 기준 변경 (최신순 등)
*   [x] 검색어 기반 모임 필터링
*   [x] URL 파라미터와 필터 상태 동기화 (새로고침 및 공유 가능)
*   [x] 로딩 상태 스켈레톤 UI 제공 (AsyncBoundary)

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 홈 화면 또는 모임 목록 페이지 접속.
*   **Step 2**: 상단 카테고리 탭에서 관심 분야 선택.
*   **Step 3**: 하단 세부 필터(레벨, 상태) 적용하여 리스트 압축.
*   **Step 4**: 정렬 기준 변경하여 원하는 순서로 조회.
*   **Step 5**: 검색창에 키워드 입력하여 특정 모임 검색.

## 5. 제외 사항 (Out of Scope)
*   지도 기반의 위치 기반 검색 (추후 단계 고려)
*   개인화된 추천 리스트 (사용자 활동 기반)
