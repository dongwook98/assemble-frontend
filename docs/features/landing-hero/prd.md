# 랜딩 히어로 (Landing Hero) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 서비스 접속 시 가장 먼저 보이는 최상단 섹션으로, 서비스의 정체성을 보여주고 사용자의 행동(모임 찾기)을 유도하는 영역.
*   **비즈니스 목적**: 신규 방문자에게 신뢰감을 주고, 즉각적인 서비스 탐색을 유도하여 이탈률을 낮춤.
*   **성공 지표 (KPI)**: '모임 찾기' 버튼 클릭률, 첫 화면 이탈률(Bounce Rate) 감소.

## 2. 사용자 스토리 (User Stories)
*   "사용자는 이 서비스가 무엇을 하는 서비스인지 첫 화면에서 바로 이해하고 싶어 한다."
*   "사용자는 복잡한 과정 없이 바로 현재 모집 중인 인기 모임들을 탐색하고 싶어 한다."
*   "사용자는 시각적으로 깔끔하고 현대적인 디자인을 통해 서비스에 대한 신뢰를 느끼고 싶어 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 눈에 띄는 메인 타이틀 및 서비스 가치 전달 문구
*   [x] 서비스의 성격을 잘 나타내는 배경 이미지 또는 일러스트
*   [x] 모임 목록으로 바로 이동할 수 있는 CTA(Call to Action) 버튼
*   [x] 검색창 통합 (선택 사항: 현재는 버튼 위주)
*   [x] 다양한 화면 크기(반응형)에 최적화된 레이아웃

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 서비스 URL 접속.
*   **Step 2**: 히어로 섹션의 문구와 이미지를 통해 서비스 파악.
*   **Step 3**: 중앙 또는 하단의 '인기 모임 보기' 또는 검색바를 통해 탐색 시작.

## 5. 제외 사항 (Out of Scope)
*   사용자별 맞춤형 히어로 컨텐츠 (차후 고도화 시 고려)
*   동영상 배경 (성능 최적화를 위해 이미지/CSS 위주로 구성)
