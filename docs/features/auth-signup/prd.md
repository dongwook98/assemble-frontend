# 회원가입 (Auth Signup) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 신규 사용자가 서비스를 이용하기 위해 계정을 생성하고 관심 카테고리를 설정하는 기능.
*   **비즈니스 목적**: 사용자 베이스 확보 및 관심사 기반의 맞춤형 추천 시스템 구축을 위한 데이터 수집.
*   **성공 지표 (KPI)**: 회원가입 전환율 (Sign-up conversion rate).

## 2. 사용자 스토리 (User Stories)
*   "사용자는 이름, 이메일, 비밀번호를 입력하여 계정을 생성할 수 있어야 한다."
*   "사용자는 비밀번호를 오입력하지 않도록 한 번 더 확인 입력할 수 있어야 한다."
*   "사용자는 본인의 관심 분야(카테고리)를 선택하여 맞춤형 정보를 제공받을 수 있어야 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 이름 유효성 검사 (2자 이상)
*   [x] 이메일 형식 유효성 검사 (Zod)
*   [x] 비밀번호 보안 정책 (8자 이상, 영문 및 숫자 조합)
*   [x] 비밀번호와 비밀번호 확인 일치 여부 검증
*   [x] 관심 카테고리 최소 1개 이상 필수 선택 (STUDY, EXERCISE, PROJECT, HOBBY, CULTURE_ART)
*   [x] 회원가입 성공 시 자동 로그인 처리 및 홈 화면 이동

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 회원가입 페이지 접속.
*   **Step 2**: 이름, 이메일, 비밀번호 입력.
*   **Step 3**: 관심 카테고리 선택 (중복 가능).
*   **Step 4**: '회원가입 완료' 버튼 클릭 (제출).
*   **Step 5**: (성공) 홈 화면으로 자동 이동 / (실패) 오류 메시지 확인.

## 5. 제외 사항 (Out of Scope)
*   이메일 중복 확인 (실제 서버 API 연동 후 보완 필요)
*   휴대폰 본인 인증
*   약관 동의 (후속 작업 필요)
