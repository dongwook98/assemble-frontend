# 로그인 (Auth Login) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 이메일과 비밀번호를 입력하여 서비스를 이용할 수 있는 인증 기능.
*   **비즈니스 목적**: 사용자 인증을 통한 개인화된 서비스 제공 및 보안 확보.
*   **성공 지표 (KPI)**: 로그인 성공률, 회원가입 전환율.

## 2. 사용자 스토리 (User Stories)
*   "사용자는 등록된 이메일과 비밀번호를 입력하여 로그인할 수 있어야 한다."
*   "사용자는 비밀번호를 잊었을 때 비밀번호 찾기 페이지로 이동할 수 있어야 한다."
*   "사용자는 계정이 없을 때 회원가입 페이지로 이동할 수 있어야 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 이메일 형식 유효성 검사 (Zod)
*   [x] 필수 항목 입력 여부 확인
*   [x] 로그인 성공 시 사용자 정보 전역 상태 저장 (Zustand)
*   [x] 로그인 성공 시 메인 페이지(HOME)로 리다이렉션
*   [x] API 오류 발생 시 적절한 알림 표시 (Alert/ApiError)

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 로그인 페이지 접속.
*   **Step 2**: 이메일 및 비밀번호 입력.
*   **Step 3**: '로그인' 버튼 클릭 (제출).
*   **Step 4**: (성공) 홈 화면으로 이동 / (실패) 오류 메시지 확인.

## 5. 제외 사항 (Out of Scope)
*   소셜 로그인 (OAuth) - 차후 단계 고려
*   자동 로그인 (Remember Me) 기능
