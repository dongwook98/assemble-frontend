# 그룹 상세 (Group Detail) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 특정 모임의 모든 정보(상세 설명, 공지, 게시판, 멤버, 일정)를 한눈에 확인하고 활동에 참여할 수 있는 통합 커뮤니티 공간.
*   **비즈니스 목적**: 모임 멤버 간의 소통을 활성화하여 서비스 활성화 지표를 높이고, 사용자의 커뮤니티 소속감을 증진시킴.
*   **성공 지표 (KPI)**: 게시글 작성 수, 참여 신청 버튼 클릭률, 평균 체류 시간.

## 2. 사용자 스토리 (User Stories)
*   "사용자는 모임의 상세 소개글과 공지사항을 확인하여 참여 여부를 결정하고 싶어 한다."
*   "사용자는 모임에 참여 신청을 하여 멤버들과 소통할 권한을 얻고 싶어 한다."
*   "사용자는 게시판을 통해 질문을 남기거나 일상적인 대화를 나누고 싶어 한다."
*   "사용자는 모임의 다가오는 일정을 확인하고 본인의 스케줄을 관리하고 싶어 한다."
*   "사용자는 모임의 멤버 구성을 확인하여 어떤 사람들과 함께 활동하는지 알고 싶어 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 모임 기본 정보 노출 (제목, 카테고리, 태그, 참여 방식, 소개글)
*   [x] 참여 신청/취소 기능 (`JoinGroupButton`)
*   [x] 찜하기/좋아요 기능 (`LikeButton`)
*   [x] 공지사항 및 일반 게시판 관리 (`GroupNotice`, `GroupBoard`)
*   [x] 참여 멤버 목록 조회 및 관리 (`GroupMembers`)
*   [x] 모임 일정 확인 및 등록 (`GroupSchedules`)
*   [x] 참여 여부에 따른 기능 제한 (비멤버는 읽기 전용 등)

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 모임 목록에서 특정 모임 카드 클릭.
*   **Step 2**: 상세 페이지 상단에서 모임 정보 및 찜하기 상태 확인.
*   **Step 3**: '가입하기' 버튼을 눌러 모임 멤버로 합류.
*   **Step 4**: 공지사항을 확인하고 게시판에 첫 인사 글 작성.
*   **Step 5**: 다가오는 오프라인 모임 일정 확인.

## 5. 제외 사항 (Out of Scope)
*   실시간 실시간 채팅 (1단계는 게시판 기반 소통)
*   멤버 강퇴 및 권한 관리 (방장 전용 기능은 차후 단계)
