# 마이페이지 (My Page) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 사용자 본인의 활동 내역(모임, 글, 댓글)을 확인하고, 프로필 정보 및 앱 설정을 관리하는 공간.
*   **비즈니스 목적**: 사용자 참여도 확인을 통한 리텐션 증대 및 개인화된 서비스 경험 제공.
*   **성공 지표 (KPI)**: 프로필 완성도, 마이페이지 재방문율, 알림 수신 동의율.

## 2. 사용자 스토리 (User Stories)
*   "사용자는 내가 가입한 모임 목록을 확인하고 해당 모임으로 바로 이동하고 싶어 한다."
*   "사용자는 내가 과거에 작성했던 게시글과 댓글을 관리하고 싶어 한다."
*   "사용자는 나의 프로필 정보(닉네임, 관심 분야 등)를 언제든 수정하고 싶어 한다."
*   "사용자는 서비스 알림(푸시/이메일) 수신 여부를 개별적으로 설정하고 싶어 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 참여 중인 모임 리스트 출력 (내가 만든 모임, 가입한 모임 구분)
*   [x] 활동 내역 조회 (작성 글, 작성 댓글)
*   [x] 프로필 수정 (이미지 업로드, 닉네임 변경 등)
*   [x] 알림 설정 (서비스 공지, 활동 알림 토글)
*   [x] 서브 네비게이션을 통한 메뉴 간 이동 (활동/그룹/설정)

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 헤더 또는 사이드바의 프로필/마이페이지 메뉴 클릭.
*   **Step 2**: 대시보드에서 최근 참여 모임 확인.
*   **Step 3**: '활동 내역' 메뉴로 이동하여 내가 쓴 글 조회.
*   **Step 4**: '설정' 메뉴에서 프로필 정보 업데이트 및 알림 옵션 조정.

## 5. 제외 사항 (Out of Scope)
*   포인트/등급 시스템 (차후 단계 고려)
*   타 사용자 프로필 조회 (공개 프로필 기능)
