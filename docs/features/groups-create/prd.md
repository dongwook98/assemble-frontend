# 그룹 생성 (Group Create) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 사용자가 본인의 관심사에 맞는 새로운 모임을 개설하고 멤버를 모집하기 위해 필요한 정보를 등록하는 기능.
*   **비즈니스 목적**: 서비스 내 신규 커뮤니티 공급을 활성화하고 사용자의 능동적인 참여(방장 활동)를 유도함.
*   **성공 지표 (KPI)**: 신규 모임 생성 수, 모임 생성 성공률, 첫 멤버 가입 기간.

## 2. 사용자 스토리 (User Stories)
*   "사용자는 본인이 운영하고 싶은 모임의 제목, 카테고리, 소개글을 상세히 작성하고 싶어 한다."
*   "사용자는 온/오프라인 여부와 활동 지역, 참여 인원 제한 등을 설정하고 싶어 한다."
*   "사용자는 모임의 성격에 맞는 이미지를 등록하여 시각적으로 매력적인 모임을 만들고 싶어 한다."
*   "사용자는 모임 생성이 완료되면 즉시 모임 상세 페이지로 이동하여 활동을 시작하고 싶어 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 모임 기본 정보 입력 (제목, 카테고리, 태그)
*   [x] 상세 활동 내용 작성 (Rich Text 또는 단순 텍스트)
*   [x] 참여 방식 설정 (온라인/오프라인, 지역 선택)
*   [x] 인원 제한 및 참여 조건 설정
*   [x] 대표 이미지 업로드 기능
*   [x] 입력 필드 유효성 검사 (Zod)
*   [x] 생성 성공 시 해당 모임 상세 페이지로 이동

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 홈 화면 또는 목록 페이지의 '모임 만들기' 버튼 클릭.
*   **Step 2**: 모임 제목 및 카테고리 선택.
*   **Step 3**: 온/오프라인 여부 및 활동 상세 규칙 작성.
*   **Step 4**: 대표 이미지 등록 및 최종 확인.
*   **Step 5**: '생성하기' 버튼 클릭 후 생성된 모임 상세 페이지 확인.

## 5. 제외 사항 (Out of Scope)
*   유료 모임 결제 기능 (차후 단계 고려)
*   방장 승인제/선착순 선택 옵션 (기본은 선착순)
