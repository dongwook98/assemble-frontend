# 비밀번호 찾기 (Auth Find Password) - Product Requirements Document (PRD)

## 1. 개요 (Overview)
*   **기능 요약**: 비밀번호를 분실한 사용자가 본인의 계정 이메일을 통해 비밀번호 재설정 링크를 받을 수 있도록 지원하는 기능.
*   **비즈니스 목적**: 사용자 이탈 방지 및 계정 보안 유지.
*   **성공 지표 (KPI)**: 비밀번호 재설정 완료율.

## 2. 사용자 스토리 (User Stories)
*   "사용자는 비밀번호를 잊었을 때 본인의 이메일을 입력하여 재설정 링크를 요청할 수 있어야 한다."
*   "사용자는 비밀번호 찾기 과정에서 다시 로그인 화면으로 돌아갈 수 있어야 한다."

## 3. 핵심 요구사항 (Core Requirements)
*   [x] 이메일 유효성 검사 (Zod)
*   [x] 비밀번호 재설정 메일 발송 안내 문구 노출
*   [x] 로그인 페이지로 돌아가기 링크 제공
*   [ ] 실제 서버 API 연동 (추후 업데이트 필요)

## 4. 사용자 여정 (User Journey)
*   **Step 1**: 비밀번호 찾기 페이지 접속.
*   **Step 2**: 가입된 이메일 입력.
*   **Step 3**: '재설정 이메일 보내기' 버튼 클릭.
*   **Step 4**: 발송 성공 안내 팝업(Alert) 확인.
*   **Step 5**: 필요시 로그인 화면으로 복귀.

## 5. 제외 사항 (Out of Scope)
*   실제 비밀번호 재설정 폼 (별도 페이지 개발 필요)
*   이메일 수신 확인 로직 (서버 사이드 영역)
